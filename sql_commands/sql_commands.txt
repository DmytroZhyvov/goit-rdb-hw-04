1.a
CREATE SCHEMA LibraryManagement;

1.b
CREATE TABLE authors (
    author_id INT AUTO_INCREMENT PRIMARY KEY,
    author_name VARCHAR(255)
 );

1.c
CREATE TABLE genres (
    genre_id INT AUTO_INCREMENT PRIMARY KEY,
    genre_name VARCHAR(255)
 );

1.d
CREATE TABLE books (
    book_id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255),
    publication_year YEAR,
    author_id INT,
    genre_id INT,
    FOREIGN KEY (author_id) REFERENCES authors(author_id),
    FOREIGN KEY (genre_id) REFERENCES genres(genre_id)
);

1.e
CREATE TABLE users (
    user_id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(255),
    email VARCHAR(255)
);

1.f
CREATE TABLE borrowed_books (
    borrow_id INT AUTO_INCREMENT PRIMARY KEY,
    book_id INT,
    user_id INT,
    borrow_date DATE,
    return_date date,
    FOREIGN KEY (book_id) REFERENCES books(book_id),
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);

4.2
SELECT COUNT(*) FROM order_details
LEFT JOIN orders ON order_details.order_id = orders.id
RIGHT JOIN customers ON orders.customer_id = customers.id
LEFT JOIN employees ON orders.employee_id = employees.employee_id
LEFT JOIN products ON order_details.product_id = products.id
LEFT JOIN categories ON products.category_id = categories.id
LEFT JOIN suppliers ON products.supplier_id = suppliers.id
LEFT JOIN shippers ON orders.shipper_id = shippers.id;

Explanation:
I modified the original query by replacing INNER joins with LEFT and RIGHT JOINs.
After that, the number of rows increased from 518 to 535.

1. This happened because RIGHT JOIN returns all rows from the right table, even if there are no matching rows in the left table.
In my case, the RIGHT JOIN was applied to the customers table:
RIGHT JOIN customers ON orders.customer_id = customers.id;

2. This means that all customers are included in the result.
If some customers do not have any related records in the orders table, they are still returned, and the columns from the orders table and related tables contain NULL values.

3. With INNER JOIN, only rows with matching values in both tables are returned, so customers without orders are excluded.

Summary: the number of rows increased because RIGHT JOIN added extra rows for customers who do not have corresponding records in the orders table.
